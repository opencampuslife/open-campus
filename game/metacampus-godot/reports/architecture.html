07GDExtension 技术选型

评估结论：有条件通过，一期范围已收敛。 原方案覆盖 6 模块 17 人日偏激进。按评估反馈调整为「构建链 PoC + 一个性能热点模块（寻路）」，前置性能基准作为迁移准入条件。对话/任务/MetricManager/TestHarness 不纳入一期。

方案对比

方案语言性能跨平台推荐
GDExtension (C++) C++ 最高，3–5× 加速【15†L27-L29】；直接访问引擎底层 Win/Mac/Linux 编译动态库，无需 .NET 运行时 ✅ 推荐
GDNative (C++) C++ 类似 仅 Godot 3.x，Godot 4 不兼容【23†L58-L60】 ❌ 废弃
Godot C# (Mono) C# 开发效率高，但与引擎交互有编组开销【21†L5960-L5964】 仅桌面平台【26†L8308-L8311】，需附带 Mono 运行时 ⚠ 备选

结论：GDExtension (C++) 是唯一符合性能目标和跨平台要求的方案。GDNative 已被 Godot 4 废弃，C# 受限于桌面平台导出和额外运行时体积。

GDExtension 工作原理

GDExtension 以动态库 (.dll/.dylib/.so) 形式加载，通过 godot-cpp 绑定层与引擎交互，无需重新编译 Godot 源码【23†L39-L46】【10†L8072-L8073】。每个扩展在 .gdextension 配置文件中声明入口符号，引擎启动时自动加载。

; example.gdextension
[configuration]
entry_symbol = "example_library_init"
compatibility_minimum = "4.3"

[libraries]
macos.debug = "res://bin/libexample.macos.debug.dylib"
windows.debug = "res://bin/libexample.windows.debug.dll"
linux.debug = "res://bin/libexample.linux.debug.so"

08模块覆写清单（评估反馈后调整）

一期：P0–P1（必做）

优先级模块重写目标语言人日入选理由
P0 GDExtension PoC + 构建链 可编译、可加载、可导出的最小扩展 C++ 3–5 先证明工程链路可用。不完成这一步，所有后续迁移无基础。
P1 性能基准框架 NPC 数量/FPS/P95 耗时/API 延迟/Smoke 耗时基线 GDScript + Python 2–3 迁移准入条件。没有基线数据，无法判断优化是否有效。必须前置。
P1 寻路系统 A* / NavMesh 路径计算
NativePathfinder 类		 C++ 5–7 最适合 C++ 的模块：边界清晰、计算密集、收益量化。多 NPC 大地图场景下 P95 帧耗时预期下降 ≥30%。

暂缓 / 不建议一期迁移

优先级模块原计划不优先的原因
P2 数据解析 / 批量规则计算 指标累加 + 存储优化 (1d) 收益太小。保留 GDScript 即可。若有大批量规则计算需求可再评估。
P3 ApiClient 网络层 HTTP + JSON + 高危拦截 (4d) 端到端延迟瓶颈在网络和 LLM 推理，不在 GDScript。优先做超时/缓存/队列优化。暂不做 C++ 覆写。
P4 对话/任务核心逻辑 DialogueManager + QuestManager (3d) 高变动业务逻辑，性能瓶颈不在语言。C++ 化会削弱 JSON 数据驱动优势和非程序人员可调性。
P4 MetricManager 指标累加 (1d) 收益极小，保留 GDScript。
P4 TestHarness HTTP 内部 HTTP 控制逻辑 (2d) 测试工具追求稳定而非性能。迁移风险 > 收益。

一期总人日：PoC (3–5) + 性能基准 (2–3) + 寻路 (5–7) = 10–15 人日（含三平台 Linux + macOS 验证，Windows 二期验收）

10构建部署 & CI/CD

构建链

  • 使用 godot-cpp 官方绑定库【10†L8078-L8083】
  • SCons: scons platform=macos target=release
  • CMake 备选方案
  • 三平台编译: .dll / .dylib / .so
  • .gdextension 配置文件声明入口符号

导出打包

  • GDExtension 仅需附带动态库文件
  • 无需 .NET/Mono 运行时
  • TestHarness + smoke 脚本保持 JSON 接口兼容
  • CI 中通过命令行启动 Godot 加载扩展后运行 smoke

GitHub Actions CI 配置

name: Build and Test GDExtension
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Install deps
        run: sudo apt-get update && sudo apt-get install -y scons build-essential git python3
      - name: Build C++ bindings
        run: cd godot-cpp && scons platform=linux target=release
      - name: Build GDExtension
        run: cd src && scons
      - name: Run Smoke Tests
        run: |
          python3 tools/smoke_g2.py
          python3 tools/smoke_g3.py
          python3 tools/smoke_g4.py

11代码示例

GDExtension (C++)

#include <godot_cpp/classes/node.hpp>
using namespace godot;

class MyNode : public Node {
    GDCLASS(MyNode, Node);
public:
    void _ready() { Godot::print("Hello from C++!"); }
    static void _bind_methods() {
        ClassDB::bind_method(D_METHOD("set_position", "pos"), &MyNode::set_position);
    }
    void set_position(const Vector2 &pos) { /* ... */ }
};

extern "C" GDNativeBool GDN_EXPORT
example_library_init(const GDNativeInterface *iface,
    const GDNativeExtensionClassLibraryPtr lib,
    GDNativeInitialization *init) {
    GDExtensionBinding::InitObject init_obj(iface, lib, init);
    init_obj.register_class<MyNode>();
    return init_obj.init();
}

12风险评估 & 验收清单

风险与缓解

学习曲线团队从 GDScript → C++。缓解：分步 PoC，保留 GDScript 基线
构建复杂度多平台 SCons/CMake。缓解：标准化 CI，Docker/VM 模拟多平台
兼容性Godot 版本升级。缓解：锁定 4.x，利用接口版本哈希【15†L64-L70】
调试难度C++ 崩溃不如脚本直观。缓解：GDB/VS 连接编辑器，充足日志
回归风险逻辑差异。缓解：全量 Smoke 测试 + 单元测试对比算法输出

量化验收清单

类别验收标准硬门槛状态
功能一致性 GDScript 与 C++ 同一输入下输出一致，核心用例 100% 通过 是 待验证
Smoke 回归 g2/g3/g4 仍为 68/68，不因 native flag 开关变化 是 ✅ GDScript 基线已通过
性能收益 目标模块 P95 帧耗时下降 ≥30%，否则不合并 是 待基准测试
稳定性 连续运行 30 分钟无崩溃、无内存持续增长 是 待验证
构建产出 CI 产出对应平台动态库 (.dylib/.so)，并能被 Godot 工程加载 是 待验证
回滚 关闭 native_enabled flag 后可完整回退到 GDScript 实现 是 待验证
可维护性 C++ API 文档、绑定说明、错误码、日志规范齐全 否 待完善

性能基准必须前置采集：NPC 数量增长下的 FPS（10/50/100/300 NPC）、寻路计算 P95/P99 耗时、主线程 frame spike 频率、API 调用端到端延迟（mock/live/fallback 分别统计）、JSON 冷/热加载耗时、Smoke 测试耗时。无基线数据不做迁移。
