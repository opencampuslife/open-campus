| **G1 Smoke Test** | 玩家移动、NPC 交互、对话框基础功能 | P0，必须通过 |

## 二、G1 Smoke Test（冒烟测试）

**目标**：验证游戏核心操作链路可用，确保评委能完成 3 分钟演示。

### TC-G1-01：玩家移动

| 字段 | 内容 |
|------|------|
| **用例编号** | TC-G1-01 |
| **测试内容** | 玩家使用 WASD/方向键在校园地图内自由移动 |
| **前置条件** | 游戏主场景加载完成，player 可见 |
| **测试步骤** | 1. 按 W 向上移动；2. 按 S 向下移动；3. 按 A 向左移动；4. 按 D 向右移动 |
| **预期结果** | 玩家角色在对应方向平滑移动，不卡顿、不穿越建筑碰撞体 |
| **验收标准** | 1. 移动响应延迟 < 100ms；2. 四方向均可达地图边界；3. 碰撞体阻止玩家穿墙 |
| **优先级** | **P0** |

### TC-G1-02：NPC 识别与显示

| 字段 | 内容 |
|------|------|
| **用例编号** | TC-G1-02 |
| **测试内容** | NPC 在地图上正确显示，玩家靠近时可识别 |
| **前置条件** | CampusMap 加载，包含 5 个 NPC |
| **测试步骤** | 1. 观察地图上 NPC 位置；2. 走近任意 NPC（碰撞区边缘） |
| **预期结果** | NPC 头顶显示角色名称标识（indicator_tint 色圆点） |
| **验收标准** | 1. 5 个 NPC 均可见；2. 靠近时触发区域高亮提示 |
| **优先级** | **P0** |

### TC-G1-03：NPC 对话触发

| 字段 | 内容 |
|------|------|
| **用例编号** | TC-G1-03 |
| **测试内容** | 玩家按 E 键触发 NPC 对话框 |
| **前置条件** | 玩家站在 NPC 交互区域内 |
| **测试步骤** | 1. 移动至 NPC 区域内部；2. 按 E 键 |
| **预期结果** | 对话框弹出，显示 NPC 名称和第一句台词 |
| **验收标准** | 1. E 键按下 < 200ms 内弹出对话框；2. 对话框不含乱码 |
| **优先级** | **P0** |

### TC-G1-04：对话框关闭

| 字段 | 内容 |
|------|------|
| **用例编号** | TC-G1-04 |
| **测试内容** | 玩家可正常关闭对话窗口 |
| **前置条件** | 对话框处于打开状态 |
| **测试步骤** | 1. 对话框打开；2. 按 Esc 键 |
| **预期结果** | 对话框关闭，焦点回到游戏主画面 |
| **验收标准** | 1. Esc 关闭延迟 < 200ms；2. 可重新按 E 再次打开同一 NPC 对话 |
| **优先级** | **P0** |

## 三、G2 对话测试（8 个 MVP 任务）

**目标**：验证 8 个 MVP 任务可完整完成，对话分支正确影响指标。

| 字段 | 内容 |
|------|------|
| **用例编号** | TC-G2-01 |
| **任务名称** | T1 - 家长招生咨询 |
| **测试内容** | 回答家长关于报名材料的问题，调用知识库并引用来源 |

| **验收标准** | 1. answer 字段非空；2. citations 数组至少 1 项；3. 指标变化在仪表盘可见 |

**目标**：验证指标变化逻辑、阈值边界、联动规则正确工作。

### TC-G3-01：学校效率变化

| 字段 | 内容 |
|------|------|
| **用例编号** | TC-G3-01 |
| **指标名称** | 学校效率（school_efficiency） |
| **测试内容** | 验证效率指标提升/下降场景的数值正确性 |
| **前置条件** | 仪表盘可访问（D 键） |
| **测试步骤** | 1. 记录初始值；2. 使用自动提醒完成任务（T3）；3. 按 D 查看效率变化 |

| **验收标准** | 1. 每次变化数值与 game-design.md §3.1 §3.2 一致；2. 仪表盘实时更新 |

|------|------|
| **用例编号** | TC-G3-02 |

## 六、测试数据规范

### 6.1 Mock 数据要求

所有测试在 mock mode 下执行，mock 数据必须包含：

1. **对话数据**（dialogues.json）：8 个任务的完整对话树
2. **任务数据**（quests.json）：8 个任务的状态和奖励定义
3. **指标数据**（metrics.json）：4 个核心指标的初始值和变化规则

### 6.2 测试账号与环境

- 无需登录账号
- 测试存档使用独立 `test_save.dat`
- 每次测试前重置为初始状态

## 七、缺陷严重等级

| 等级 | 定义 | 示例 |
|------|------|------|
| **Blocker** | 阻塞发布，评委无法完成演示 | 移动失效、无法打开对话框 |
| **Critical** | 核心功能失效 | 指标不变化、任务无法完成 |
| **Major** | 功能异常但可绕行 | NPC 显示缺失、颜色偏差 |
| **Minor** | 视觉或文案问题 | 文字错别字、图标错位 |

## 八、验收条件

1. G1 冒烟测试 4/4 通过（全部 P0）
2. G2 对话测试 6/8 通过（T1-T5, T7，6 个 P0 必过）
3. G3 指标测试 3/4 通过（效率、信任、合规 + 联动）
4. 无 Blocker 级缺陷残留
