## 一、规格定义（Spec Definitions）

### 1.1 核心规格

| 资源类型 | 规格 | 备注 |
|----------|------|------|
| **Tile 尺寸** | 32x32 像素 | 地图基础单元 |
| **角色尺寸** | 32x32 像素 | 玩家与 NPC 统一尺寸 |
| **角色动画** | 四方向，每方向 3 帧 | 上/下/左/右，共 12 帧 |
| **地图大小** | 40x30 tiles | 1280x960 像素（@32x32） |
| **UI 分辨率** | 1280x720 | 默认基准 |
| **UI 适配** | 支持 1920x1080 | 可选全高清适配 |

### 1.2 像素艺术风格原则

- **低复杂度像素风**：不追求高精度美术，保持简洁清晰的像素风格
- **色彩饱和度适中**：避免过于鲜艳或过于暗淡的颜色
- **轮廓清晰**：确保每个元素在俯视角下可辨识
- **动画帧数精简**：角色 3 帧/方向，UI 过渡无需动画
- **禁止使用未经授权的商业素材**：所有资源必须原创或使用开源许可

## 四、TileMap 层定义（TileMap Layer Structure）

| 层级 | Layer 名称 | 碰撞 | 内容示例 |
|------|------------|------|----------|
| **Layer 0** | Ground | 无碰撞 | 草地、土地、道路 |
| **Layer 1** | Buildings | 有碰撞 | 教学楼、办公室、食堂 |
| **Layer 2** | Interactive | 无碰撞 | 门、可交互物体 |
| **Layer 3** | Decoration | 无碰撞 | 树木、花草、装饰 |
| **Layer 4** | NPC | 无碰撞（触发器） | NPC 角色 |
| **Layer 5** | Overlay | UI 遮罩 | 阴影、特殊效果 |

### 4.1 Tile 命名规范

```
tiles/
├── ground/
│   ├── grass_tile.png      # 草地
│   ├── path_tile.png       # 道路
│   └── floor_tile.png      # 室内地板
├── buildings/
│   ├── wall_tile.png       # 墙体
│   ├── door_tile.png       # 门
│   ├── window_tile.png     # 窗户
│   └── roof_tile.png       # 屋顶
├── decoration/
│   ├── tree_tile.png       # 树木
│   ├── flower_tile.png     # 花草
│   └── bench_tile.png      # 长椅
└── special/
    ├── highlight_tile.png  # 高亮可交互
    └── shadow_tile.png      # 阴影
```

### 4.2 Tile 设计要点

| 类别 | 像素密度 | 颜色数 | 特点 |
|------|----------|--------|------|
| Ground | 低 | 2-3 色 | 简单纹理，便于拼接 |
| Buildings | 中 | 3-4 色 | 清晰轮廓，功能区隔 |
| Decoration | 低 | 2 色 | 纯装饰，无功能 |
| Special | 可变 | 可变 | 交互提示用 |

## 五、角色 Sprite 规范（Character Sprite Specification）

### 5.1 角色帧结构

```
characters/
├── player/
│   ├── player_down_32x32.png   # 下（默认朝向）
│   ├── player_up_32x32.png     # 上
│   ├── player_left_32x32.png   # 左
│   ├── player_right_32x32.png  # 右
│   └── player_shadow.png       # 阴影
├── npc/
│   ├── parent_32x32.png        # 家长 NPC
│   ├── teacher_32x32.png       # 老师 NPC
│   ├── logistic_32x32.png      # 后勤 NPC
│   ├── ai_assistant_32x32.png  # AI 助手
│   └── student_32x32.png       # 学生 NPC
└── effects/
    ├── interact_icon.png       # 交互提示图标
    └── quest_marker.png        # 任务标记
```

### 5.2 角色设计要点

| 元素 | 规格 | 说明 |
|------|------|------|
| 角色大小 | 28x28 px（留 4px 阴影） | 中心位置 |
| 阴影 | 4px，透明度 30% | 角色下方椭圆阴影 |
| 轮廓 | 1px 深色描边 | 增强辨识度 |
| 颜色 | 角色对应主色为主 | 辅助色 1-2 种 |
| 动画 | 逐帧像素动画 | 3 帧/方向，步行循环 |

### 5.3 各角色视觉特征

| 角色 | 体态 | 服装 | 标识 |
|------|------|------|------|
| 玩家 | 标准立姿 | 蓝色制服/西装 | 胸前 AI 徽章 |
| 家长 | 站立等候 | 休闲装/正装 | 手提包 |
| 老师 | 站姿专业 | 教师制服 | 手持文件夹 |
| 后勤 | 忙碌姿态 | 工作服/马甲 | 工具包 |
| AI 助手 | 悬浮/站立 | 科技感服装 | 发光标识 |
| 学生 | 活泼站姿 | 校服 | 背包 |

## 八、素材制作规范（Asset Creation Guidelines）

### 8.1 禁止事项

- **禁止使用未经授权的商业素材**：所有资源必须原创或使用开源许可
- **禁止使用真实学生照片**：角色均为像素风格非写实
- **禁止使用第三方图标库商业版**：仅使用开源/免费图标
- **禁止高分辨率写实美术**：保持低复杂度像素风

### 8.2 推荐工具

| 用途 | 工具 | 格式 |
|------|------|------|
| 像素绘制 | Aseprite | .ase / .png |
| 图标制作 | Piskel（在线） | .png |
| UI 设计 | Figma | 导出 .png |
| 纹理生成 | Tiled | .tsx |

### 8.3 导出规范

| 类型 | 格式 | 压缩 | 命名 |
|------|------|------|------|
| Tiles | PNG | 无压缩 | `{name}_32x32.png` |
| Characters | PNG | 无压缩 | `{name}_{direction}_{index}.png` |
| Icons | PNG | 无压缩 | `{name}_24x24.png` 或 `{name}_32x32.png` |
| UI Elements | PNG | 无压缩 | `{name}_ui.png` |
