# P4.5 Controlled Internal Pilot Plan

## Objective

Validate the Gaokao Agent system with real admissions counselors
in a controlled internal environment before any public launch.

## Scope

| What | Scope |
|------|-------|
| Counselors | 5-10 internal admissions counselors |
| Sessions | 30-50 real/semi-real parent/student consultations |
| Knowledge documents | 10-20 real documents ingested |
| Remote URL ingestion | DISABLED |
| Public website chat | NOT enabled |
| WeChat Work integration | NOT enabled |
| Enterprise WeChat | NOT connected |

## Pre-Pilot Freeze

```bash
git tag pilot-p4.5-internal-v1
make release-check
make backup
make recovery-drill
```

## Pilot Accounts

| Role | Count | Purpose |
|------|-------|---------|
| admin | 1 | System monitoring, audit review, dangerous actions |
| campus_admin | 1-2 | Knowledge publishing, campus-scoped views |
| sales | 5-10 | Parent consultation, lead follow-up, session takeover |
| content_operator | 1 | Knowledge ingestion, staging docs |
| reviewer | 1 | Content review and approval |

## Pilot Stages

### Day 1-2: Counselor Onboarding
- Login to admin console (`/admin/dashboard`)
- Review knowledge base content
- Run test consultations with known queries
- Session takeover practice
- Lead creation and follow-up practice
- Feedback form: "What's missing?"

### Day 3-5: Active Usage
- Counselors handle parent/student consultations
- Sales team reviews sessions, takes over where appropriate
- Content operator ingests real knowledge documents
- Reviewer approves/rejects staging content
- Admin monitors audit logs and health endpoints

### Day 5+: Review and Iterate
- Collect all feedback
- Fix knowledge gaps
- Adjust mode routing if needed
- Improve tone quality
- Expand FSM if needed
- Prepare for wider pilot

## Monitoring Metrics (Daily)

| Metric | Target | Alert Threshold |
|--------|--------|-----------------|
| Chat sessions | > 20/day | < 5 (low usage) |
| Handoffs | > 3/day | > 15 (too many) |
| New leads | > 5/day | 0 (no conversion) |
| Emotional support triggers | — | > 40% (too many emotional) |
| Tone quality failures | < 5% | > 15% |
| Compliance blocks | 0 | > 3 |
| Rate limit hits | < 10 | > 50 |
| CSRF failures | 0 | > 5 |
| Source leaks detected | 0 | > 0 |
| Crisis escalations | — | Any (review immediately) |
| Sales follow-up rate | > 60% | < 30% |

## Daily Review Sample (10 conversations)

```
5 parent consultations
5 student consultations  
All handoff conversations
All crisis escalations
All emotional support mode conversations
```

## Knowledge Gap Tracking

During pilot, track:
- Questions that couldn't be answered
- Topics where counselors had to correct the AI
- Common parent/student questions not in the KB
- Phrases the AI used that felt unnatural

## Exit Criteria

- [ ] 100+ conversation samples collected
- [ ] No permission/security leaks
- [ ] No admin/internal content exposed to parent/student
- [ ] No promise/compliance violations in production
- [ ] No source lines visible to parent/student
- [ ] Emotional support mode rated acceptable by counselors
- [ ] Sales can follow up based on lead summaries
- [ ] At least 1 round of KB gap fixes applied
- [ ] `make release-check` PASS continuously

## Go/No-Go Gates

After Day 5:

| Gate | Condition |
|------|-----------|
| GO → Wider Pilot | All exit criteria met, counselor approval |
| HOLD → More Iteration | KB gaps found, tone issues, fix then re-test |
| NO-GO → Roll Back | Security leaks, compliance failures, unacceptable quality |
